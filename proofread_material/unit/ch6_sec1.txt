¥section{Definitions for Turing Machines}

A Turing machine stores data on an infinite tape, each position
(also called a ``cell'')
on the tape can hold a single symbol from an alphabet‾$¥Gamma$.
The tape contents is accessed with a tape head that can read from
or write into a single cell at a time. The tape head
can move
a single step leftward or rightward, one cell at a time.
In addition to the tape contents, a Turing machine maintains
a current ``state'' value; there are only finitely
many possible states.
The action of the machine is controlled by a ``finite control''
or ``transition function'' that describes the action
of the Turing machine based on the current state and the current
symbol being read by the tape head.

¥index{Turing machine!definition of|(}
Turing machines have severe restrictions on how they
can access data on their tape, since the contents of only one tape cell
can be read at a time and since the tape head
can only move one cell at a time. Modern-day computers
can, of course, access data in much more flexible ways,
especially with the use of random access memory (RAM) to directly
access data by its address in memory. Nonetheless, setting
aside issues of efficiency, a Turing
machine can perform any operation that can be done by
a computer. The point is that Turing machines were designed to
be a minimalistic model of computation that still
captures all possible effective computations.

The key features of a Turing machine are as follows:
¥begin{enumerateLoose}
¥item The Turing machine stores its data on an infinitely long,
linear tape. The tape consists of cells, each cell contains a single
symbol from the ¥idxem{tape alphabet}‾$¥Gamma$.¥footnote{Multitape Turing
machines can use finitely many tapes.}
¥item The Turing machine reads one cell at a time from the tape.
The current cell being read is called the ¥idxem{tape head position}.
The tape head can move one cell left or right at a time.
¥item There is an alphabet‾$¥Sigma$ called the ¥idxem{input alphabet}.
We have $¥Sigma ¥subsetneq ¥Gamma$. There is a special ¥idxem{blank symbol}‾$¥#$
which is in‾$¥Gamma$ but not in‾$¥Sigma$. A Turing machine is initialized
with only finitely many non-blank symbols on its tape.  If the Turing
machine takes a single input‾$w¥in¥Sigma^*$, it is initialized
with the input‾$w$ written in consecutive tape cells and the rest of
the tape cells holding blank symbols‾$¥#$. A Turing machine that
takes $k$ inputs $w_1,¥ldots, w_k$ is initialized with the
string $w_1¥#w_2¥# ¥cdots ¥# w_{k-1}¥#w_k$ and the rest
of the tape blank. The tape head position starts at the
first (leftmost) symbol of its first input.¥footnote{%
Definition‾¥ref{def:TMinputConfig} gives the formal
definition of how inputs are given to a Turing machine.}
¥item The Turing machine has a ¥idxem{finite control} which consists of a
finite set‾$Q$ of ¥emph{states}¥index{state}, and a finite list of instructions
for how the Turing machine should act. These instructions are
encoded with a ¥idxem{transition function}‾$¥delta$.
¥item At any given step, the Turing machine is in a particular state‾$q¥in Q$,
called the ¥idxem{current state}, and is reading the symbol‾$a ¥in ¥Gamma$
in the tape cell under the tape head. The value $¥delta(q,a)$ gives
the instructions on what symbol should be written in the current tape cell,
whether to move the tape head left or right one square, and what the next
state should be.
¥item There is
a designated ¥idxem{start state} (or ``initial state'') often denoted‾$q_0$;
the Turing machine starts running in its start state.
¥item There is a set‾$Q_¥Thalt$
of designated halting¥index{halting state} states. A Turing machine that is used
to decide or semidecide a relation has two halting states,
often denoted $q_¥Tacc$ and‾$q_¥Trej$, that are designated accepting
and rejecting states. A Turing machine that computes or partial computes
a function or enumerates a set has a single designated ¥idxem{output state},
often denoted‾$q_¥Tout$. When the
output state is entered, the output string is equal to the maximum
length string‾$w$ that is contained in the tape cells starting
at the current tape cell and going rightward to the first symbol
that is in $¥Gamma ¥setminus ¥Sigma$. Generally, we impose the further condition
that a string‾$w$ is output when $¥# w ¥#$ appears on the tape with the
tape head positioned over the first symbol of‾$w$.  Similar conventions
are adopted for a Turing machine that outputs $k$-tuples of strings in‾$¥Sigma^*$
to enumerate a $k$-ary relation.¥footnote{%
Definition‾¥ref{def:acceptReject} formally defines how a Turing
accepts and rejects strings (or $k$-tuples of strings).
Definitions ¥ref{def:outputConvention}
and‾¥ref{def:outputConventionMulti} formally define what it means for
a Turing machine to output a string or a $k$-tuple of strings.}
¥end{enumerateLoose}

The next definition formalizes the description of a Turing machine as
an 8-tuple $(¥Gamma, ¥Sigma, Q, q_0, ¥delta, Q_¥Thalt, q_¥Tacc, q_¥Trej)$
or 7-tuple $(¥Gamma, ¥Sigma, Q, q_0, ¥delta, Q_¥Thalt, q_¥Tout )$
¥begin{definition}
A Turing machine with accept/reject states is specified by an 8-tuple
$(¥Gamma, ¥Sigma, Q, q_0, ¥delta, Q_¥Thalt, q_¥Tacc, q_¥Trej )$
such that
¥begin{enumerateLoose}
¥item[(a)] $¥Gamma$ and‾$¥Sigma$ are the ¥idxem{tape alphabet}
and the ¥idxem{input alphabet}. We have $¥Sigma ¥subseteq ¥Gamma$;
the ¥idxem{blank symbol} is a member of $¥Gamma ¥setminus ¥Sigma$.
¥item[(b)] $Q$ is a finite set of ¥emph{states}¥index{state}.
¥item[(c)] $q_0 ¥in Q$ is the ¥idxem{start state}.
¥item[(d)] There are two distinct designated accept and reject states
$q_¥Tacc, q_¥Trej ¥in Q$.
¥item[(e)] $Q_¥Thalt = ¥{q_¥Tacc, q_¥Trej¥}$ is the
set of ¥emph{halting¥index{halting state} states}.
¥item[(f)] The ¥idxem{transition function}‾$¥delta$ is
a function $¥delta : (Q¥setminus Q_¥Thalt) ¥times ¥Gamma ¥rightarrow  ¥Gamma ¥times ¥{¥TL,¥TR¥} ¥times Q$.
Thus, when $q$ is a non-halting state and $a$‾is a tape alphabet symbol,
$¥delta(q,a)$ is equal to a triple $(a', ¥TL, q^¥prime)$ or $(a', ¥TR, q^¥prime)$
where $a^¥prime$‾is a tape alphabet symbol and $q^¥prime$‾is a state.
The symbols $¥TL$ and‾$¥TR$ mean ``move Left'' and ``move Right''.
¥end{enumerateLoose}
¥end{definition}

¥begin{definition}
A Turing machine that outputs strings is specified by a 7-tuple
$(¥Gamma, ¥Sigma, Q, q_0, ¥delta, Q_¥Thalt, q_¥Tout )$
such that $¥Gamma$, $¥Sigma$, $Q$, $q_0$ and‾$¥delta$ are as above and
such that
¥begin{enumerateLoose}
¥item[(d${}^¥prime$)] $q_¥Tout¥in Q$ is the ¥idxem{output state}.
¥item[(e${}^¥prime$)] The set $Q_¥Thalt$ of ¥emph{halting states}¥index{halting state}
is equal to either $¥{ q_¥Tout ¥}$ or the empty set‾$¥varnothing$.
¥end{enumerateLoose}
¥end{definition}

The meaning of the ¥idxnm{transition function} is that
$¥delta ( q, a )$ gives the instructions on what the Turing machine
should do when in state‾$q$ reading the symbol‾$a$.  The values
¥[
¥delta(q, a) ‾= ‾ (a', ¥TL, q^¥prime) ¥qquad¥hbox{and}¥qquad
¥delta(q, a) ‾= ‾ (a', ¥TR, q^¥prime)
¥]
give the instructions that when in state‾$q$ reading the symbol‾$a$,
the Turing machine should overwrite $a$ with‾$a^¥prime$,
move the tape head one cell to the left or right (respectively),
and then enter state‾$q^¥prime$.
¥index{Turing machine!definition of|)}

A ¥idxem{configuration} (also called an `instantaneous description'')
of a Turing machine is a complete
description of the Turing machine at a given step: this includes specifying the
contents of the tape, the current state, and the current position of the
tape head. A computation of a Turing machine thus consists of
a sequence of configurations; the 
initial¥index{configuration!initial} configuration
is in the initial state (usually called‾$q_0$) and has the input string
or strings written on the tape starting at the tape head position.
(See Definition‾¥ref{def:TMinputConfig} below.)
The final configuration (if any) is in a halting state and is
called a halting¥index{configuration!halting} configuration.
(Definitions ¥ref{def:acceptReject},
¥ref{def:outputConvention} and‾¥ref{def:outputConventionMulti}
describe how a Turing machine returns results.)
It is possible that the
Turing machine runs forever without entering a halting configuration.

A Turing machine
transitions from one configuration to the next as specified
by the transition function; see Definition‾¥ref{def:nextConfig} below.

¥begin{example}¥label{example:TuringEvenLength}
For a first, simple example, we construct
a Turing machine that accepts exactly inputs
of even length. We'll use the input alphabet $¥Sigma = ¥{1¥}$
and the tape alphabet $¥Gamma = ¥{1, ¥#¥}$. The action
of the machine is informally described as follows. It alternates
between two states $q_0$ and‾$q_1$
while scanning the input string from left to right.
It maintains the invariant that it is in state $q_0$ or‾$q_1$ depending
on whether an even or odd number of 1's have been seen so far. When
the first blank symbol‾($¥#$) is reached, the machine enters either
$q_¥Tacc$ or‾$q_¥Trej$.

Figure‾¥ref{fig:TuringEvenLengthExample}

¥begin{eqnarray*}

¥delta( q_1, 1 ) &=& (1, ¥TR, q_0) ¥¥
¥delta( q_0, ¥# ) &=& (¥#, ¥TL, q_¥Tacc) ¥¥

¥end{eqnarray*}

The Turing machine above
accepts (that is, decides) the set
$¥{ w ¥in¥penalty10000 ¥Sigma^* : ¥hbox{$|w|$ has even length} ¥}$.
The next definitions make this formal.
¥begin{definition}
¥index{Turing machine!input string(s)}¥label{def:TMinputConfig}%
¥index{configuration!initial}
Let $M$ be a Turing machine with input alphabet‾$¥Sigma$.
The machine‾$M$ is ¥emph{given $w¥in ¥Sigma^*$ as input} if
it is started in state‾$q_0$, the tape is entirely blank other than
the sequence of consecutive tape cells containing the string‾$w$, and
the tape head position is over the first (leftmost) symbol of‾$w$, or over
a blank symbol if $w = ¥epsilon$.

The machine‾$M$ is ¥emph{given a $k$-tuple $w_1,¥ldots, w_k ¥in ¥Sigma^*$ as input}
if it is started in state‾$q_0$, the tape is entirely blank apart from
a sequence of consecutive tape cells containing the string‾$w_1¥#w_2¥#¥cdots ¥# w_k$, and
the tape head position is over the first (leftmost) symbol of‾$w_1$, or if $w_1 = ¥epsilon$,
it is over
the blank symbol where $w_1$ would have started.
¥end{definition}

¥begin{definition}¥label{def:acceptReject}
Let $M$ be a Turing machine with accept and reject states. Fix
$k¥ge 1$. We say $M$‾¥emph{accepts}¥index{accept}
a $k$-tuple $w_1,¥ldots, w_k$, if, when it is given
$w_1,¥ldots, w_k$ as input, it eventually enters
its accepting state. If it eventually enters its rejecting
state, then we say $M$‾¥emph{rejects}¥index{reject}
$w_1,¥ldots, w_k$.
¥end{definition}

Note that it is possible that $M$ neither accepts nor rejects‾$W$ and instead
runs forever without halting.

¥begin{definition}¥label{def:languageTM}
The notation $L(M)$ denotes the set of strings‾$w$ such that
$M$ accepts‾$w$. This is called the ¥idxem{language} of‾$M$.
¥end{definition}

¥begin{definition}¥label{def:TuringDecides}
A subset‾$R$ of‾$¥Sigma^*$ is ¥idxem{Turing semidecidable}
if there is a Turing machine‾$M$ such that $L(M) = R$.
Note that, when given an $w ¥notin R$, $M$‾may either reject‾$w$
or may run forever without halting.
We also say that $M$ ¥emph{semidecides}¥index{semidecide}‾$R$.

If there is a Turing machine‾$M$ such that $L(M) = R$
and such that $M$ halts on all inputs, then
we say that $R$ is ¥idxem{Turing decidable}.
In this case, $M$ ¥emph{decides}¥index{decide}‾$R$.
¥end{definition}

The above Example ¥ref{example:TuringEvenLength}
shows that the set of even strings
over $¥Sigma = ¥{1¥}$ is Turing decidable.  By the
¥idxnm{Church-Turing Thesis},
being Turing semidecidable is the same as being
semidecidable (and the same as being computably
enumerable). Similarly, being Turing decidable is the
same as being decidable.

¥begin{example}¥label{example:TuringComplement}
Let $¥Sigma = ¥{0,1¥}$ and define the ¥idxnm{binary complement} function‾$f$ be the
unary function such that $f(w)$ is obtained from‾$w$ by interchanging 0's and‾1's
in‾$w$. For example, $f(1011) = 0100$.
A Turing machine‾$M$ that computes‾$f$ is as follows: The tape alphabet
is $¥Gamma = ¥{0,1,¥#¥}$ and the states are $Q = ¥{q_0, q_1, q_¥Tout¥}$.
The machine‾$M$ starts
at the first symbol of‾$w$, and first
scans from left to right overwriting 0's with‾1's, and 1's with‾0's.
When it reaches the first blank symbol, it scans back to the
beginning of‾$w$. It does this by scanning leftward until reaching
a‾$¥#$, and then moving one tape cell rightward before ending
up in the output state‾$q_¥Tout$. When $M$ halts, the tape head is
positioned over the first symbol of the output string.

The machine‾$M$ uses state‾$q_0$ to scan
rightward, and then state‾$q_1$ to scan leftward.  The transition function is
given by
¥begin{multicols}{2}

¥delta( q_0, 1 ) &=& (0, ¥TR, q_0) ¥¥
¥delta( q_0, ¥# ) &=& (¥#, ¥TL, q_1)
¥end{eqnarray*}
¥noindent
¥begin{eqnarray*}
¥delta( q_1, 0 ) &=& (0, ¥TL, q_1) ¥¥
¥delta( q_1, 1 ) &=& (1, ¥TL, q_1) ¥¥
¥delta( q_1, ¥# ) &=& (¥#, ¥TR, q_¥Tout)
¥end{eqnarray*}
¥end{multicols}
¥noindent
and $q_¥Tout$ is a halting state, so $Q_¥THalt = ¥{q_¥Tout¥}$.

The above example gives a Turing machine that computes
the binary complement function. This is formalized in the next
definitions.
¥begin{definition}[Outputting a string]¥label{def:outputConvention} ¥index{Turing machine!output string(s)}
Let $M$ be a Turing machine with a halting output
state‾$q_¥Tout$ and with input alphabet‾$¥Sigma$.
When (and if) $M$ enters‾$q_¥Tout$, the output string is the
string $v ¥in ¥Sigma^*$ written on the tape, with the tape head
positioned at the first (leftmost symbol) of‾$v$, and with $v$ terminated
on the tape with a member of‾$¥Gamma ¥setminus ¥Sigma$. Generally,
we require w.l.o.g.¥ that $v$‾be terminated with‾$¥#$.¥footnote{The reason for
allowing $w$ to be terminated by a symbol from $¥Gamma ¥setminus ¥Sigma$ other than
the blank symbol‾$¥#$ is that we want the output string to be
a well-defined member of‾$¥Sigma^*$ whenever $M$ enters its output state.}
¥end{definition}
¥begin{definition}¥label{def:TuringComputes}
Let $M$ and $k$ be as above. Then $M$ ¥emph{partial computes} the
function‾$f$ such that, for all $w_1,¥ldots,w_k ¥in ¥Sigma^*$,
¥begin{itemizeLoose}
¥item
If $f(w_1,¥ldots,w_k)¥converges = v$, then the the machine‾$M$ given
$w_1,¥ldots,w_k ¥in ¥Sigma^*$ as
input eventually halts with output‾$v$;
¥item
If $f(w_1,¥ldots,w_k)¥diverges$ then $M$ given $w_1,¥ldots,w_k ¥in ¥Sigma^*$ as
input never halts.
¥end{itemizeLoose}
We say that $f$ is ¥idxem{Turing partial-computable}.
Note that $f$ is uniquely determined by $M$ and‾$k$.

If $f$ is total, then we say $M$ ¥emph{computes}‾$f$ and that
$f$ is ¥idxem{Turing computable}.
¥end{definition}
By the ¥idxnm{Church-Turing Thesis}, a function‾$f$ is computable
if and only if it is Turing computable; and it is
partial computable if and only if it is Turing partial-computable.

We have so far defined the notions of ``Turing semidecidable'', ``Turing decidable'',
and ``Turing (partial) computable''. We next give an example of an
``Turing enumerable'' set.

¥begin{example}¥label{example:TuringEvenEnumerate}
Let $¥Sigma = ¥{1¥}$ and consider the
set $R = ¥{ 1^n: ¥hbox{$n$ is even} ¥}$.
We construct a Turing machine‾$M$ that enumerates‾$R$.
Let the tape alphabet be $¥Gamma = ¥{¥#, 1¥}$,
let the set of states be $Q = ¥{q_0, q_1, q_2 ¥}$,
let the start state be‾$q_0$,
let the output state (usually called‾$q_¥Tout$) also
be‾$q_0$, and let $Q_¥Thalt = ¥varnothing$ so there is
no halting state.  Define the transition function‾$¥delta$
so that
¥begin{multicols}{2}
¥noindent
¥begin{eqnarray*}
¥delta( q_0, ¥# ) &=& (1, ¥TL, q_1) ¥¥
¥delta( q_0, 1 ) &=& (1, ¥TL, q_0)
¥end{eqnarray*}
¥noindent
¥begin{eqnarray*}
¥delta( q_1, ¥# ) &=& (1, ¥TL, q_2) ¥¥
¥delta( q_2, ¥# ) &=& (1, ¥TR, q_0)
¥end{eqnarray*}
¥end{multicols}
¥noindent
The values of $¥delta(q_1,1)$ and $¥delta(q_2,1)$ are
immaterial for the action of‾$M$ when started on
a completely blank tape, since $M$‾will never
read a‾1 while in state $q_1$ or‾$q_2$. We therefore omit
specifying those values of‾$¥delta$.

The machine‾$M$ produces infinitely many outputs and never halts.
It acts by repeatedly adding two‾1's to the front
of the string on the tape and outputting the result. The states
$q_0$ and‾$q_1$ are used to add the‾1's, moving the tape head leftward
with each symbol. The state‾$q_2$ is used to step back to
output a string in state‾$q_0$.  As shown in
Figure‾¥ref{fig:TuringEvenEnumerateExample},
the strings that are output are‾$¥epsilon$,
then‾$11$, then‾$¥epsilon$, then $1111$, etc.
The fact that the empty string‾$¥epsilon$ is repeatedly
output is OK since a Turing machine is permitted to
output duplicate strings when enumerating a set.¥footnote{The
duplicate outputs could be eliminated by using a four state

The machine‾$M$ is said to ``¥idxnm{enumerate}'' the
set‾$R$. This is formalized in the next definitions.
The first definition gives the conventions for
how a Turing machine outputs a $k$-tuple of strings.
It coincides with Definition‾¥ref{def:outputConvention}
when $k=1$.
¥begin{definition}[Outputting a $k$-tuple of strings]¥index{Turing machine!output string(s)}
¥label{def:outputConventionMulti}
Let $M$ be a Turing machine with a non-halting output
state‾$q_¥Tout$ and with input alphabet‾$¥Sigma$, and let $k¥ge¥penalty10000 1$.
Whenever $M$ enters‾$q_¥Tout$, it outputs a $k$-tuple
$¥langle w_1,¥ldots, w_k¥rangle$
of strings from‾$¥Sigma^*$. By convention, the output strings are written
on the tape in the form $v = w_1 ¥#_1 w_2¥#_2 ¥ldots ¥#_{k-1} w_k ¥#_k$
where each $w_i¥in¥Sigma^*$ and each $¥#_i ¥in ¥Gamma ¥setminus ¥Sigma$,
and the tape head is positioned on the first symbol of‾$w_1$ or
on the symbol‾$¥#_1$ if $w_1 = ¥epsilon$.
The symbols $¥#_i$ do not need to be the blank symbol‾$¥#$; however,
it is the usual convention that the‾$¥#_i$ are all equal to‾$¥#$.
¥end{definition}

When working with a Turing machine with a non-halting
output state, we fix a value for $k$ so that the machine is
viewed as outputting $k$-tuples of strings.

¥begin{definition}¥label{def:TuringEnumerates}
Let $M$ and $k$ be as above. Then $M$ ¥emph{enumerates}¥index{enumerate}
the $k$-ary relation‾$R$ containing exactly the $k$-tuples that are
output by‾$M$.  The relation may contain finitely many or
infinitely many tuples. We say that $R$ is ¥idxem{Turing enumerable}.
¥end{definition}

By the ¥idxnm{Church-Turing Thesis}, $R$‾is Turing enumerable
if and only if it is computably enumerable.

At this point, it should be clear how a Turing machine
operates. But, for the sake of completeness, we give the
formal definition of how a Turing machine transitions from one configuration
to the next:

¥begin{definition} ¥label{def:nextConfig}
Suppose that a Turing machine is in a configuration‾$C$ such
that (a)‾the machine is in a non-halting state‾$q$;
(b)‾the tape contains the string $vaw$ where $v,w¥in ¥Gamma^*$,
$a ¥in¥Gamma$, $|v|¥ge 1$, $|w|¥ge 1$, and the rest of the tape is blank (contains only
the symbols‾$¥#$); and (c)‾the tape head is positioned over
the symbol‾$a$.  The next configuration‾$C^¥prime$ that follows‾$C$
in the Turing machine's computation is
defined by:
¥begin{itemize}
¥item If $¥delta(q, a) = (a', ¥TL, q')$, then $C^¥prime$‾has $va'w$ written on the tape
in place of $vaw$, is in state‾$q'$, and has the tape head positioned
over the last symbol of‾$v$.
¥item If $¥delta(q, a) = (a', ¥TR, q')$, then $C^¥prime$‾has $va'w$ written on the tape
in place of $vaw$, is in state‾$q'$, and has the tape head positioned
over the first symbol of‾$w$.
¥end{itemize}
¥end{definition}
